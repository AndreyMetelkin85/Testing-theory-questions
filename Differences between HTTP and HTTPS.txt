    Протокол HTTP...
 
 Проток HTTP или HyperText Transfer Protocol это главный протокол сервиса интернет
 WWW(всемирная паутина). Основная задача протокола, обеспечить передачу гипертекста в сети.
 В протоколе описывается формат сообщений, для обмен клиентов и серверов. 
 
 URL... 
 
 Основой веб-общения клиент - сервер является запрос. Запрос отправляется при помощи URL - единого
 указателя ресурсов Интернет. Напоминаю, что такое URL адрес.
 
 http://www.example.com:1234/put/do/resursa?c=b&x=y
 
 Где:
 
 http:// - Протокол
 www.example.com - хост
 1234 - Порт
 /put/do/resursa - путь до ресурса
 ?c=b&x=y - запрос
 
Понятная и простая структура URL состоит из следующих элементов:
 
   Протокол 
   Хост
   Порт
   Каталог ресурса
   Метки(запрос).
   
Примечание: Протокол http это протокол для простых, не защищенных соединений.
Защищённые соединения работают по протоколу https. Он более безопасен для 
обмена данными.


Как работает HTTPS...

HTTPS — это протокол безопасной передачи данных, поддерживает технологию шифрования TLS/SSL.
Стандартный протокол HTTP передает данные в открытом виде. Злоумышленники могут
вклиниться в передачу изменить или перехватить данные. В HTTPS для передачи данных создаётся 
защитный канал. Вот как это происходит:

- Васин браузер посылает запрос к сайту

- сайт отправляет в ответ копию сертификата

- браузер проверяет подлинность сертификата - узнает у центра сертификации, который его выдал

- если сертификат не поддельный, сайт и браузер тайно договариваются о симметрическом ключе

С помощью этого ключа Васин браузер и сайт устанавливают защищённое HTTPS-соединение.
Ключ шифрует данные - мошенники не могу получить доступ у паролям и номерам кредиток пользователя.

Простыми словами:

HTTP – открытый протокол передачи данных, а HTTPS – закрытый, имеющий надстройку шифрования
Данные протоколы использую разные порты http порт 80 https порт 443.
В https применяться шифрование данных, которые затем расшифровываются с помощью ключей на сервере.
